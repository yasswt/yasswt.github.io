《结网》读书笔记

发表于 2017-12-03 | 分类于 读书笔记

互联网产品经理

1、定义 ：通常是负责对现有互联网产品进行管理及营销的人员，也负责开发新产品。

2、工作内容 ：

  • 战略性工作（15%～25%）：为产品建立长期的战略布局；发现新的产品机会；为产品的演变、增强和引进提供建议。
  • 阶段性工作（20%～30%）：参与新产品开发；参与年度商业计划的制定；利用公司内部、外部资源开展营销活动；预测竞争对手的行动并制定应对方案；更新产品并进行相应的用户教育；降低成本；重新规划产品线。
  • 日常性工作（40%～55%）：收集分析竞争情报；协调开发、运营、客服、销售等资源以保证产品正常运作；执行商业计划。

3、Point ：产品经理在面对各种难题的时候，第一反应不应该是简单的做选择，而要先跳出来，看清楚这到底是不是选择题，相信如何创造用户价值。

互联网产品

1、创造步骤：用户价值（概念）—— 产品形态（形式）—— 产品定位（风格）—— 信息架构&HCI设计（结构）—— 技术研发&项目管理（工艺）—— GUI设计（外观）。

2、获取产品概念的方式：

  • 受现有产品的启发，将既有概念进行转换。
  • 出于自身需求或捕捉到了他人的需求，而这个需求尚未被满足。
  • 预见用户需求的变化，提出超前的原创概念。

3、Web 2.0 ：如何更好的满足用户的需求

Web 1.0 Web 2.0
不列颠百科全书在线 Wikipedia
个人网站 blogging
目录（分类法） tagging（“民间分类”）
DoubleClick Google AdSense

4、蓝海战略：同时追求“差异化”和“低成本”，从而实现价值创新。

  • 成功与否取决于一款产品能不能在它的蓝海时段内达到理想的用户规模或筑起足够高的壁垒。
  • 开启时间非常重要。

5、互联网广告：搜索广告、展示类广告、分类广告、引导广告、电子邮件。

6、现金牛产品：利润超高的产品，并且利润占公司总利润的很大比例。

7、平台产品：能通过自身的资源优势拉动其他产品的产品。

  • 导入用户数 = 平台产品的活跃用户数 x 转化率 x 拉动时长。
  • 提升转化率前提：平台要对用户产生影响，让他们去使用被拉动的产品，给予被拉动产品一定的展示面积和展示时间是非常重要的。
  • 先发后至效应：具有平台产品的公司可以等其他公司的概念有了实验结果后再复制到自己的平台上，利用自己的用户规模和触达用户的能力让后发的产品快速超越先发的产品。

8、焦点小组：一次性邀请6～9个具有代表性的用户对某一主题或观念进行深入讨论的方法。

  • 群体动力学指出，只要有别人在场，一个人的思想行为就同他单独一个人时有所不同，会受到他人影响，6～9个用户所呈现的结果是具有普遍性的，并非个人看法。
  • 用户的历史行为（真实意愿）比他们的当前意愿更有价值。

9、可用性测试步骤：寻找一些具有代表性、符合产品潜在用户条件的用户。

  • 请这些具有代表性的用户在产品或产品原型上完成一些任务。
  • 使用体验观察室观察他们的操作，他们在哪里成功了，在哪里卡住了，倾听他们的意见。

10、网站产品设计文档：

  • 网站结构图：描述整个网站的结构，确定了网站的模块划分及网页个数。
  • 网站线框图：描述一个网页中所需要包含的基础元素及其在这个页面中的布局。
  • 网页描述表：包含一个网页中所需要展示的元素及其背后的逻辑。

11、用户体验：别让我等！别让我想！ 别让我烦！

  • 别让我等：用户耐心有限。
    • 绝对速度一定要及格，在此基础上越快越好。
    • 用户体验影响产品口碑，口碑影响产品成败，产品成败影响产品经理的利益。
    • 如果完整性的牺牲处于可容忍的范围内，用完整性来换取速度、响应和服务器开销是最好的选择。
  • 别让我想：
    • 包装好网站：LOGO + Slogan = 辨识度。
    • 关注用户及其任务，给予明确的指引。
    • 尽可能降低用户学习成本。
  • 别让我烦：缩短完成路径，提高操作效率。

12、网站分析：通过对用户行为进行研究，为网站改进提供决策依据。

  • 原则：可行动 - 促进网站改进；以用户为中心。
  • 网站分析系统：页面标记 & 日志分析。
    • 建立并维护业务数据模型；
    • 对产品更新、营销活动等进行专项严重，评估效果，总结经验；
    • 监护产品的健康情况，确保产品完成KPI；
    • 对产品的关键指标进行预测，为商业计划提供数据支持。
  • 方法：细分 & 对比。
  • 产品运营状况简报：KPI完成状况与预测。产品计划与执行情况。

13、拉动：

  • 平台拉动：
    • 转化率：平台展示面积；平台用户和被拉动产品目标用户重叠度；封闭体验。
    • 植入：将产品入口放到平台中；把产品的品牌或展示放到某个内容中；将产品的“品牌”或“入口”放到用户“身”上。
  • 口碑拉动：硬指标 + 差异化 = 品牌（tag）

14、更新：

  • 管理用户期望要素：更新本身 & 更新频率。
  • 种类：功能更新、内容更新、热点事件应激反应等。
  • 更新功能优先级：将更新内容按照指标和差异化分两类列出；分析每个更新的成本和效应，排列在平面坐标系中；位于低成本高效益的象限中的更新优先考虑。

15、优雅降级：对产品运行环境进行分析，让产品在不健全的环境中通过有损运行的方式来保持部分核心体验的方法；本质是取舍；可用性和”别让我等“是底线。

理论及方法

1、“拙劣的艺术家模仿，伟大的艺术家偷取” ——毕加索

2、马太效应：如果几位科学家在同一时期发现了同一个定律，功劳通常会归属于最有名那位科学家，不知名的科学家往往都会被无视掉。这样一来，受到过肯定的科学家就会“做出”越来越多的贡献，真正的第一发现者往往都被埋没了。

3、NIH综合症：人们不愿意使用、购买或接受某种产品、研究成果或者知识，不是出于技术或者法律等因素，而只是因为它源自其他地方。

4、MECE原则（麦肯锡）：相互独立，完全穷尽。

  • 在实际网站结构分析中可以使用面向对象的方法。
  • 用户闭环：一系列帮助用户完成任务的环节，这些环节可以应对任务可以出现的各种情况。

5、消灭拖延提升效率：把任务分解为连续的、在能力范围之内的小任务。

6、墨菲定律网站版：

  • 凡是输入框，都会遭遇灌水、SPAM、脚本注入；
  • 凡是积分，都会被刷；
  • 凡是推到网站首页的内容，都会出现色情，政治；
  • 凡是用户间沟通的渠道，都会被广告机器人利用。

7、墨菲定律项目版：

  • 一项工作如果只有一个人负责，这个人肯定会休假或离职。
  • 认为没有技术难点的地方，都会成为技术难点或性能瓶颈。

8、危机管理：加强信息披露和与用户的沟通，争取用户的谅解和支持。

9、奥卡姆剃刀定律：如无必要，勿增实体。

10、CE：Costomer Engagement

  • 提供一个明显的反馈入口，引导用户发表反馈和投诉，并及时回复处理。
  • 订阅产品关键字的搜索结果，了解官方反馈论坛之外的反馈，给予反馈和处理。
  • 建立或加入核心用户群，第一时间获取他们的想法，也可以很方便的测试一些产品概念。
  • 通过用户行为录像工具或跟用户回家的方法，追踪单个用户的使用行为，发现解决使用障碍。
  • 对流失用户进行回访，分析流失原因，改进产品降低流失率。

11、封闭体验：如果用户使用A产品的体验叠加上使用B产品的体验，相对于使用A产品加B产品的竞品可以获得增值或低成本，那么我们可以说A产品和B产品的组合形成了封闭体验。
